# BookTG — мини-апп «личная библиотека + подписка»

Telegram Mini App, где пользователь загружает свои книги (.txt, .epub и .pdf), читает их,
а платная подписка через **Telegram Stars** снимает лимит на число книг и открывает
премиум-функции. Прогресс и подписка хранятся на сервере (привязаны к Telegram-аккаунту).

Есть два источника книг:
- **Моя библиотека** — каждый пользователь загружает СВОИ книги (приватно, видны только ему).
- **Каталог** — общие книги, которые добавляешь ты как владелец; их видят все, а чтение
  открывается по подписке (можно отдельные книги пометить бесплатными).

> ⚖️ Важно: в каталог можно добавлять только то, что разрешено распространять —
> общественное достояние (Чехов, Толстой и т.д.) или книги с лицензией/разрешением автора.
> Добавлять туда современные защищённые книги нельзя — это пиратство и блокировка бота.

## Возможности интерфейса

- Светлый современный дизайн, цветные обложки книг, экран «Продолжить чтение» с кольцом прогресса.
- Чтение **.txt**, **.epub** (epub.js) и **.pdf** (pdf.js).
- Панель «Aa» прямо во время чтения: 4 темы (Свет / Крем / Сепия / Ночь),
  размер и тип шрифта, межстрочный интервал, яркость. Настройки сохраняются между сессиями.
- Прогресс чтения хранится на сервере и виден на полке и в карточке «Продолжить».

## Что внутри

- `server.js` — бэкенд (Node.js + Express): авторизация через Telegram, загрузка книг,
  библиотека, подписка Stars и вебхук оплаты.
- `public/index.html` — само приложение (библиотека, загрузка, читалка TXT/EPUB, экран подписки).
- `package.json`, `.env.example`.

Хранилище для MVP — простой JSON-файл (`data/db.json`) и папка `data/uploads`.
Без баз данных и внешних сервисов, чтобы запустить было максимально просто.

## Что понадобится

1. **Компьютер** — развернуть бэкенд с телефона не получится.
2. Аккаунт на **Render.com** (есть бесплатный тариф) или Railway.
3. Аккаунт **GitHub** (код удобно деплоить из репозитория).
4. **Токен бота** от `@BotFather` (тот же бот, что уже создан).

## Шаг 1. Локальный запуск (по желанию)

```bash
npm install
BOT_TOKEN=ваш_токен npm start
# откроется на http://localhost:3000
```

Внутри Telegram приложение проверяет подпись `initData`, поэтому в обычном браузере
без Telegram оно покажет «Откройте в Telegram» — это нормально.

## Шаг 2. Залить код на GitHub

Создайте репозиторий и загрузите туда все файлы проекта (без папки `node_modules`).

## Шаг 3. Деплой на Render

1. Render → **New** → **Web Service** → подключите свой GitHub-репозиторий.
2. Настройки:
   - **Build Command:** `npm install`
   - **Start Command:** `npm start`
3. В разделе **Environment** добавьте переменные из `.env.example`
   (как минимум `BOT_TOKEN` и `WEBHOOK_SECRET`).
4. Нажмите **Create Web Service**. Render выдаст адрес вида
   `https://booktg-xxxx.onrender.com`.

## Шаг 4. Включить оплату (вебхук)

Чтобы сервер узнавал об оплатах, один раз привяжите вебхук бота к своему адресу.
Откройте в браузере (подставьте свой токен, адрес и секрет):

```
https://api.telegram.org/bot<ВАШ_ТОКЕН>/setWebhook?url=https://booktg-xxxx.onrender.com/webhook/<ВАШ_WEBHOOK_SECRET>
```

Должно прийти `{"ok":true,...}`.

## Шаг 5. Указать приложение в BotFather

В `@BotFather`:
- `/myapps` → ваш web app → отредактируйте **Web App URL** на адрес Render
  (`https://booktg-xxxx.onrender.com`), **либо** заведите новый через `/newapp`.
- Для кнопки в чате: `/setmenubutton` → тот же адрес.

Готово — откройте бота, загрузите книгу, проверьте оплату.

## Как наполнять каталог

1. Узнайте свой Telegram ID у бота `@userinfobot`.
2. Впишите его в переменную `ADMIN_IDS` на хостинге (можно несколько через запятую).
3. Откройте приложение — на вкладке **Каталог** у вас появится кнопка
   «Добавить в каталог». Загружайте туда книги (галочка «бесплатно» — если книга
   должна открываться без подписки).
4. Обычные пользователи видят каталог, но платные книги открываются только по подписке.

## Как работают деньги

- Цифровые подписки в Telegram продаются только за **Telegram Stars (XTR)**.
- Подписка рекуррентная: Telegram сам списывает Stars каждые 30 дней и шлёт серверу
  уведомление, по которому премиум продлевается.
- Комиссия: при покупке Stars в мобильных приложениях Apple/Google берут ~30%
  (итого ~32% на мобильных, ~3–4% на десктопе). Вывод заработка — через Fragment в TON.
  Закладывайте это в цену.

## Ограничения MVP и что улучшить дальше

- **Хранилище эфемерное.** На бесплатном Render файлы и `data/db.json` могут сбрасываться
  при передеплое/засыпании. Для продакшена:
  - база → **PostgreSQL** (Render/Neon/Supabase);
  - файлы книг → объектное хранилище (**S3 / Cloudflare R2 / Supabase Storage**).
- Поддерживаются .txt, .epub и .pdf. PDF поддерживается через pdf.js.
- Нет админки и аналитики — добавляется по мере роста.

## Идеи премиум-функций (легальные)

Синхронизация между устройствами, безлимит книг, озвучка текста (TTS),
статистика чтения, словарь/перевод по тапу, экспорт цитат, расширенные темы.
